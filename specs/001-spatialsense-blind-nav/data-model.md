# Data Model: SpatialSense — Blind Navigation Assistant

**Phase**: 1  
**Date**: 2026-04-30

---

## Entities

### Session

The lifecycle container for a single user interaction: one image + one question → one spatial report. No state persists between sessions.

| Field | Type | Description |
|-------|------|-------------|
| `session_id` | str | UUID generated at query start |
| `image` | PIL.Image | Original RGB camera frame |
| `question` | str \| bytes | User's question — text transcript or raw audio bytes |
| `depth_tensor` | torch.Tensor \| None | Shape `(1,1,H,W)`, metric meters, float32. None until a TIPSv2 tool call completes. |
| `seg_mask` | torch.Tensor \| None | Shape `(H,W)`, int. ADE20K class index per pixel. None until a TIPSv2 tool call completes. |
| `intrinsics` | CameraIntrinsics | Focal length and principal point; extracted from EXIF at session creation or estimated from 65° FOV fallback |
| `depth_colormap` | PIL.Image \| None | Colormap image — demo display artifact only, never sent to Gemma |
| `overlay` | PIL.Image \| None | ADE20K segments colored by mean depth, alpha-blended onto RGB — demo display artifact only, never sent to Gemma |
| `spatial_report` | SpatialReport \| None | The final response generated by Gemma |

**Validation rules**:
- `depth_tensor` and `seg_mask` are None until `call_dpt_head` or `call_encoder_zero_shot` completes; `measure_object` must not be called before they are populated
- `image` must be a valid RGB image (3-channel); grayscale images are converted to RGB before processing
- Session lifetime is one request/response cycle; depth tensor and seg mask are released after the response is returned

---

### Query

The unit of user interaction: an image paired with a question.

| Field | Type | Description |
|-------|------|-------------|
| `image` | PIL.Image | Camera frame representing the current environment |
| `question_text` | str \| None | Typed text question (secondary path — testing and dev only) |
| `question_audio` | bytes \| None | Raw audio bytes of spoken question (primary path) |

**Validation rules**:
- At least one of `question_text` or `question_audio` must be non-null
- `image` is mandatory; queries without an image are rejected with a natural language error message

---

### SpatialReport

The structured response generated by Gemma, containing the complete navigation description.

| Field | Type | Description |
|-------|------|-------------|
| `text` | str | Full natural language response in plain language |
| `audio` | bytes | TTS-synthesized audio of the response |
| `observations` | list[ObjectObservation] | One entry per identified object; ordered closest to farthest |
| `confidence` | str | `"high"` \| `"medium"` \| `"low"` — overall confidence in the spatial analysis |
| `fallback` | bool | True if the response is a fallback (object not found, low image quality, or error) |

---

### ObjectObservation

One identified object within a SpatialReport.

| Field | Type | Description |
|-------|------|-------------|
| `label` | str | Object name (ADE20K class name or zero-shot class name) |
| `distance_m` | float | Approximate distance in meters from the camera |
| `bearing_deg` | float | Horizontal bearing in degrees from camera center to the object centroid; negative = left of center, positive = right; computed from pixel centroid + session camera intrinsics |
| `confidence` | str | `"high"` \| `"medium"` \| `"low"` — confidence in this specific observation |
| `zero_shot` | bool | True if located via zero-shot encoder path rather than DPT segmentation head |

**Confidence rules** (sourced from `measure_object` return value):
- `"low"` when `distance_m` is near 0.001m or 10.0m (TIPSv2 depth clip extremes)
- `"low"` when fewer than 50 segmentation mask pixels fall within the bounding box (object barely visible or box misplaced)
- `"medium"` when the zero-shot path was used (cosine similarity mask is coarser than argmax segmentation)
- `"high"` when a clean segmentation mask was found within the box and depth is within the reliable range (0.5m – 8m)

**Note on bearing_deg**: `bearing_deg` accuracy depends on camera intrinsics quality. When `session.intrinsics.source == "assumed"`, bearing is approximate (±5° error typical for standard smartphones). When `source == "exif"`, bearing is derived from measured focal length.

---

### DepthMap

The per-pixel metric depth representation, cached per session.

| Field | Type | Description |
|-------|------|-------------|
| `tensor` | torch.Tensor | Shape `(1,1,H,W)`, float32, values in meters |
| `width` | int | Frame width in pixels |
| `height` | int | Frame height in pixels |

**Depth range**: TIPSv2 DPT head clips output to [0.001m, 10.0m]. Values near 0.001m or 10.0m indicate low-confidence readings and should trigger cautious language ("I can see something very close, but I'm not confident about the exact distance").

**Reliable range for SC-002**: 0.5m – 8.0m. Distance accuracy within ±20% of ground truth is the acceptance criterion for objects in this range.

---

### CameraIntrinsics

Camera focal length and principal point, used to compute horizontal bearing angles from pixel positions. Extracted once at session creation.

| Field | Type | Description |
|-------|------|-------------|
| `fx` | float | Horizontal focal length in pixels |
| `fy` | float | Vertical focal length in pixels |
| `cx` | float | Principal point x (horizontal image center, pixels) |
| `cy` | float | Principal point y (vertical image center, pixels) |
| `source` | str | `"exif"` when derived from EXIF 35mm equivalent focal length; `"assumed"` when EXIF absent |

**EXIF derivation** (iPhone and most cameras):
```
crop_factor  = focal_35mm_eq / focal_mm
sensor_width = 36 / crop_factor
fx = (focal_mm / sensor_width) * image_w
fy = (focal_mm / sensor_width) * image_h
cx = image_w / 2
cy = image_h / 2
```

**Fallback** (when EXIF absent or PNG input): assume 65° horizontal FOV → `fx = image_w / (2 * tan(32.5°))`, `cx = image_w / 2`, `cy = image_h / 2`. Bearing accuracy degrades to ±5° typical.

**Bearing computation** (in `measure_object`):
```
px, py = centroid of (seg_mask == class_idx) & pixels_inside_box
bearing_deg = degrees(atan2((px - cx) / fx, 1))   # depth cancels; bearing is depth-independent
```

## State Transitions

```text
Query submitted (RGB image + audio or text question)
    │
    ▼
Session created
  intrinsics populated ← extract_intrinsics(image) — EXIF path or 65° FOV fallback
  depth_tensor = None
  seg_mask = None
    │
    ├── Gemma calls search_seg_classes(query)
    │       Substring match on ADE20K class list
    │       returns: {"matches": [...]}
    │       Session state unchanged
    │
    ├── Gemma calls call_dpt_head(image_ref)
    │       TIPSv2 DPT forward pass
    │       depth_tensor populated ← (1,1,H,W) float32 meters
    │       seg_mask populated ← (H,W) int, ADE20K class index per pixel
    │       depth_colormap and overlay generated → stored for demo display, NOT sent to Gemma
    │       returns: {"status": "ready", "detected_classes": [...]} to Gemma
    │
    ├── Gemma calls call_encoder_zero_shot(image_ref, class_list)  [alternative path]
    │       TIPSv2 base encoder + text encoder
    │       depth_tensor populated ← same contract as call_dpt_head
    │       cosine similarity map cached as object mask
    │       returns: {"status": "ready", "detected_classes": [...]} to Gemma
    │
    ├── Gemma draws bounding box around target object instance on the RGB
    │   Gemma calls measure_object(class_name, box_2d)
    │       Precondition: depth_tensor and seg_mask are not None
    │       Intersects: (seg_mask == class_idx) & pixels inside box_2d
    │       Computes median depth over intersection pixels → distance_m
    │       Computes pixel centroid of intersection → bearing_deg via session.intrinsics
    │       returns: {"distance_m": float, "bearing_deg": float, "confidence": "high"|"medium"|"low"}
    │   Gemma calls measure_object again for each additional instance
    │
    ▼
Gemma generates SpatialReport.text
    │
    ▼
TTS synthesizes SpatialReport.audio
    │
    ▼
Session ends
  depth_tensor released from memory
  Response (text + audio + overlay) returned to Gradio UI
```

## ADE20K Class Vocabulary

TIPSv2's segmentation head outputs logits for 150 ADE20K classes. The full list is returned by `get_seg_classes()`. Navigation-critical classes present in the vocabulary:

**Indoor navigation**: wall, floor, ceiling, door, table, chair, sofa, bed, window, stairs, column, cabinet, shelves, counter, sink, bathtub, toilet, lamp, curtain, mirror, shelf, desk, wardrobe, picture/painting, person

**Outdoor navigation**: building, sky, road, grass, tree, sidewalk, car, bus, truck, bicycle, traffic light, pole, signboard, fence, railing

**Not in vocabulary** (zero-shot path required): shopping cart, step/curb, puddle, escalator handrail, turnstile, bollard
